## Post Anatomy

```
[ХУК] ─────────── Первая строка, scroll-stopper (самое важное!)
    │
[ИНТРИГА] ─────── Зацепка для перехода дальше
    │
[ТЕЛО] ────────── Раскрытие темы (1-10 веток по 500 символов)
    │
[CTA] ──────────── Призыв к действию
    │
[ССЫЛКА] ───────── ТОЛЬКО в закрепленном комментарии!
```

---

## Технические ограничения

| Параметр | Значение |
|----------|----------|
| Символов в ветке | 500 max |
| Количество веток | Неограничено |
| Видео | До 5 минут |
| Картинки | Да, повышают конверсию |
| Ссылки в посте | Режутся алгоритмами! |

## 8 Типов контента (по эффективности)

### 1. Вопросы (для раскачки аккаунта)

**Когда использовать:** На старте, для вовлечения, каждый день

**Структура:**
```
[Контекст/Пример]
[Вопрос, на который ХОЧЕТСЯ ответить]
```

**Примеры:**
- "Психологи, расскажите, как вы пришли в профессию?"
- "Какие у вас денежные ритуалы? Кофе с корицей? Денежная девятка на руке?"

**Важно:** НЕ агрессивные вопросы! "Почему у вас до сих пор нет..." - плохо.

---

### 2. Лайк-посты / Пожелания

**Когда использовать:** Каждый день, утром - отлично раскачивают охваты

**Структура:**
```
[Пожелание/Утверждение]
Лайк, если [согласен/хочешь]
```

**Примеры:**
- "Самый приятный звук — уведомление о поступлении денег. Лайк, если хочешь слышать его чаще"
- "Ни лицо, ни фигура — глубина принятия себя. Лайк, если согласны"
- "Желаю тебе сегодня встретить человека, который изменит твою жизнь"

**Эффект:** Создает образ доброго эксперта, на которого хочется подписаться.

---

### 3. Личные истории / Внутренний конфликт

**Когда использовать:** Для вовлечения и доверия

**Структура:**
```
[Ситуация из прошлого]
[Проблема/Конфликт]
[Как разрешилось / Что поняла]
[Вывод/Мораль]
```

**Примеры тем:**
- Недооцененность ("Клиентка оценила меня по кухне")
- Страх ("Я заблокировала всех друзей, когда решила уйти из найма")
- Ошибка ("Я не могла продавать, потому что...")
- Преодоление ("У меня случилась проблема со спиной, и мечта о сцене...")

**Важно:** История должна показывать вас с ХОРОШЕЙ стороны!

---

### 4. Результаты и кейсы (особенно для твердых ниш)

**Когда использовать:** Когда есть что показать

**Структура:**
```
[Эмоциональный хук] Боже/Ура/Шок!
[Результат с цифрами]
[Как достигли / Короткая история]
[CTA]
```

**Примеры:**
- "Боже, у меня первая продажа консультации с Threads!"
- "Моя клиентка забеременела после 5 лет попыток!"
- "134 подписчика за день с одного поста"

**Важно:** Подавайте ЭМОЦИОНАЛЬНО. "Моя клиентка заработала 100к" vs "БОЖЕЧКИ, моя клиентка заработала первые 100к, это что-то нереальное!"

---

### 5. Польза / Лайфхаки

**Когда использовать:** Для экспертности, перевода в Telegram

**Структура:**
```
[Хук с обещанием]
1. [Пункт]
2. [Пункт]
3. [Пункт]
...
[CTA → в закреп]
```

**Примеры:**
- "5 скрытых функций ChatGPT, о которых молчат 99%"
- "7 техник, которые оживляют любой текст"

**Нюанс:** В твердых нишах польза залетает РЕДКО. В мягких — лучше.

---

### 6. Афоризмы от эксперта

**Когда использовать:** Для мягких ниш, каждый день

**Структура:**
```
Как [профессия] официально заявляю:
[Смелое/Поддерживающее утверждение]
```

**Примеры:**
- "Как психолог официально заявляю: вы не сломаны"
- "Как маркетолог скажу: деньги в рекламе — не главное"

---

### 7. Астрология / Нумерология (для мягких ниш)

**Когда использовать:** Если релевантно нише — ЗОЛОТО

**Структура:**
```
[Категория людей по дате/знаку]
[Что их ждет / Какие они]
[Если не нашли свое → закреп / Telegram]
```

**Примеры:**
- "Люди, родившиеся 7, 16, 25 числа — вас ждет успех в июне"
- "Эти знаки выйдут замуж за миллионеров"
- "Что принесет транзит Урана? Овен: [...]"

**Фишка:** Тех, чьих чисел нет — переводить в Telegram на разбор.

---

### 8. Метафорические карты

**Когда использовать:** Для мягких ниш — супер вовлечение

**Структура:**
```
Выбери карту [описание]
[Варианты в комментариях или → Telegram]
```

**Важно:** НЕ выдавайте все карты в Threads! Переводите в Telegram.

## Структуры длинных постов

### PAS (Problem-Agitate-Solve)
```
Ветка 1: [Проблема] — боль аудитории
Ветка 2: [Усиление] — что будет, если не решить
Ветка 3-4: [Решение] — ваш метод
Ветка 5: [CTA] → закреп
```

### Story Arc
```
Ветка 1: [Завязка] — ситуация + интрига
Ветка 2: [Развитие] — что произошло
Ветка 3: [Кульминация] — поворот
Ветка 4: [Развязка] — чем закончилось
Ветка 5: [Мораль + CTA]
```

### List Format
```
Ветка 1: [Хук] X способов/ошибок/правил
Ветка 2: 1-2 пункта
Ветка 3: 3-4 пункта
Ветка 4: 5+ пункта
Ветка 5: [CTA] → закреп
```

## CTA Types

| Тип CTA | Когда использовать | Пример |
|---------|-------------------|--------|
| **В закреп** | Основной | "Подробнее в закрепе" |
| **Вопрос** | Для вовлечения | "А как у вас? Делитесь!" |
| **Лайк** | Для охватов | "Лайк, если согласны" |
| **Подписка** | Для роста | "Подписывайся, чтобы не пропустить" |
| **В Telegram** | Для перелива | "Забирай в Telegram" |

## Где ставить ссылку

| Место | Можно? | Результат |
|-------|--------|-----------|
| В посте | НЕТ | Режется алгоритмами, охваты падают |
| В закрепленном комментарии | ДА | Оптимально |
| В шапке профиля | ДА | Обязательно |
| В обычном комментарии | ДА | Работает, но хуже закрепа |

**Важно:** Только ОТКРЫТЫЕ ссылки на Telegram! Закрытые не открываются с телефона.

## Скользкая горка (по Халилову)

**Главный принцип:** Задача первого предложения — заставить прочитать второе.

```
Хук → Интрига → Тело → CTA
      ↓
Каждый элемент передаёт "эстафету любопытства"
Читатель не может остановиться
```

### Элементы скользкой горки в Threads

| Элемент | Как работает | Пример в Threads |
|---------|--------------|------------------|
| **Сцепление** | Первая строка цепляет | "134 подписчика с одного поста" |
| **Интрига** | Открытая петля | "Но главное я скажу в конце..." |
| **Твист** | Неожиданный поворот | "А вот здесь я ошибался..." |
| **Конкретика** | Цифры, факты | "За 21 день, 732 подписчика" |

### Пружины горки

- **Упругость** — нет воды, каждое слово на месте
- **Визуальность** — текст создаёт картинку
- **Эмоция** — не оставляет равнодушным
- **Конкретика** — есть что "унести" и пересказать

## Writing Principles

| Принцип | Как применять |
|---------|---------------|
| **Одна идея = один пост** | Не размывай |
| **Короткие предложения** | Мобильное чтение |
| **Переносы строк** | Визуальное дыхание |
| **Разговорный стиль** | Пиши как говоришь |
| **Конкретика > Абстракция** | Цифры, имена, детали |
| **Эмоции** | Особенно в кейсах |
| **Главный герой — читатель** | Не о себе, о НИХ |
| **Будь за белых или за красных** | Чёткая позиция, без полутонов |
| **Текст для пересказа** | Можно рассказать другу за 30 сек |
| **Никаких канцеляризмов** | Живой язык |

## Writing Workflow

```
1. Начни с хука (потрать 50% времени)
2. Напиши тело — одна четкая мысль
3. Добавь CTA
4. Сократи на 30%
5. Прочитай вслух
6. Проверь первую строку — видна в ленте?
7. Ссылку — в закрепленный коммент
```

## Твердые vs Мягкие ниши

### Твердые (маркетологи, SMM, продюсеры)

| Что работает | Что не работает |
|--------------|-----------------|
| Результаты с цифрами | Общая польза |
| Эмоциональные кейсы | Провокации (много хейта) |
| Внутренний конфликт | Шаблонные посты |
| Лайфхаки | Копирование чужих постов |

**Нюанс:** Нет шаблона — нужно думать самому. Посты быстро выгорают.

### Мягкие (психологи, эзотерики, коучи)

| Что работает | Особенности |
|--------------|-------------|
| Вопросы на широкую | Раскачка за 5 дней |
| Лайк-посты | Каждый день |
| Астрология/нумерология | Золотая жила |
| Метафорические карты | Супер вовлечение |
| Афоризмы | Легко масштабировать |

**Нюанс:** Можно использовать шаблоны, повторять — выгорают медленнее.

## Anti-patterns

| Ошибка | Почему плохо | Как исправить |
|--------|--------------|---------------|
| Ссылка в посте | Режут охваты | Только в закреп-коммент |
| Провокации/хейт | Не приносят подписчиков | Добрые посты работают лучше |
| Мешанина форматов | Сбивает образ | Выбери тон и держи |
| Нытье | Отталкивает | Истории преодоления |
| Заходить расстроенным | Напишешь лишнего | Остынь, потом пиши |
| Пиар под чужими постами | Выглядит дешево | Давай ценность в комментах |

## Quick Checklist перед публикацией

### Базовое
- [ ] Хук цепляет за 1-2 секунды?
- [ ] Есть интрига для чтения дальше?
- [ ] Пост раскрывает ОДНУ идею?
- [ ] Есть CTA?
- [ ] Ссылка в ЗАКРЕПЛЕННОМ комментарии?
- [ ] Ссылка ОТКРЫТАЯ (не частный канал)?
- [ ] Пост создает образ доброго эксперта?

### Скользкая горка (по Халилову)
- [ ] Первое предложение заставляет читать второе?
- [ ] Есть "эстафета любопытства" между ветками?
- [ ] Текст упругий (нечего убрать без потери смысла)?
- [ ] Есть конкретика (цифры, факты)?
- [ ] Можно пересказать другу за 30 секунд?
- [ ] Главный герой — читатель (не вы)?
- [ ] Нет канцеляризмов?
- [ ] Занята чёткая позиция?

## Посты про инструменты/стеки (вирусный формат)

**Контекст:** Посты про инструменты набирают 10-20K views. Правильный формат критичен.

### Структура, которая залетает

```
[Хук: Мой стек для X в YYYY:]

1. [Инструмент] ([цена]) — [для чего]: [конкретные задачи через запятую]
2. [Инструмент] ([цена]) — [метафора что это]
3. [Инструмент] ([условия покупки]) — [главная выгода]

[Итог с конкретной выгодой: сумма + что даёт]

[Полный вопрос к аудитории]
```

### Пример (18.7K views)

```
Мой стек для работы в 2026:

1. Claude Max (100$/мес) — для всего: тексты, код, анализ
2. Notion (10$/мес) — второй мозг и CRM
3. MacWhisper (один раз купил навсегда) — говорю вместо печати

Всего $110/мес за систему, которая экономит часы каждый день.

Что в вашем стеке?
```

### Ключевые правила

| Элемент | Как правильно | Как неправильно |
|---------|---------------|-----------------|
| Список | **Нумерация 1. 2. 3.** | Тире (–) |
| Пояснение | **Развёрнутое**: "для всего: тексты, код, анализ" | Краткое: "тексты" |
| Цена | **С /мес или условием**: "100$/мес", "один раз навсегда" | Просто число: "$100" |
| Итог | **Конкретная выгода**: "экономит часы каждый день" | Абстракция: "и голова" |
| Вопрос | **Полный**: "Что в вашем стеке?" | Обрезанный: "А вы?" |

### Почему работает

1. **Нумерация** — создаёт ощущение системы
2. **Развёрнутые пояснения** — показывает экспертизу
3. **Конкретная сумма** — читатель может примерить на себя
4. **Выгода в итоге** — отвечает на "зачем мне это"
5. **Вопрос** — приглашает в комменты
