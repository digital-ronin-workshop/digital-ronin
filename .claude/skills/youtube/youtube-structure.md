# YouTube Script Structure

## Когда использовать
- Планирование структуры видео
- Написание payoffs (тела контента)
- Построение логики повествования
- Работа с топ-листами

---

## Логлайн (Пиши ДО сценария)

**Логлайн = суть видео в 1-2 предложениях**

```
Формула: [КТО] + [ЧТО ДЕЛАЕТ/УЗНАЕТ] + [ЗАЧЕМ/РЕЗУЛЬТАТ]
```

Примеры:
- "Девушка узнает 3 шага, чтобы перестать быть удобной"
- "Эксперт показывает, как за копейки избавиться от купероза"

**Правило:** Не можешь написать логлайн — не понимаешь о чём видео.

---

## Форматы по длине

| Формат | Длина | Характеристики |
|--------|-------|----------------|
| Short | 2-5 мин | Quick tips, без глубины |
| Medium | 6-12 мин | Tutorials, 3-5 payoffs |
| Long | 15+ мин | Deep dives, 5+ payoffs |

**150 слов ≈ 1 минута spoken**

---

## Базовая структура

```
HOOK (50-100 слов, ~30 сек)
↓
PAYOFF 1 (300-500 слов)
↓
PAYOFF 2 (300-500 слов)
↓
...
↓
MEGA PAYOFF (самый важный)
↓
CTA (50-100 слов)
```

---

## Структура Payoff

**Payoff = момент, когда награждаешь зрителя за досмотр**

### Компоненты payoff:
1. **Setup** — введи тему, создай anticipation
2. **Development** — раскрой с логикой, примерами
3. **"So what?"** — почему это важно зрителю
4. **Zinger** — punchy one-liner в конце
5. **Open loop** — переход к следующему payoff

### 2 типа payoffs:
| Тип | Описание | Когда |
|-----|----------|-------|
| Emotional | Заставляет чувствовать | Documentary, storytelling |
| Informational | Раскрывает информацию | Educational, how-to |

---

## Payoff Structures (шаблоны)

### 1. Problem → Solution → Proof
```
SETUP: "Most people [PROBLEM]"
DEVELOP: "Here's why: [EXPLANATION]"
SOLUTION: "Instead, do [SOLUTION]"
PROOF: "For example, [CASE]"
ZINGER: "[ONE-LINE SUMMARY]"
```

### 2. Mistake → Why Wrong → Fix
```
SETUP: "Mistake #[N]: [MISTAKE]"
WHY WRONG: "This is problematic because [REASON]"
EXAMPLE: "For example: [BAD EXAMPLE]"
FIX: "Here's how to fix it: [SOLUTION]"
```

### 3. Framework Reveal
```
SETUP: "I've created [FRAMEWORK] for [PURPOSE]"
OVERVIEW: "It has [N] parts: [LIST]"
DEEP DIVE: Each part gets mini-payoff
SUMMARY: "[FRAMEWORK] = [COMPONENTS]"
```

### 4. Story → Lesson
```
SETUP: "Let me tell you about [PERSON]"
CONTEXT: "[BACKGROUND]"
CONFLICT: "[CHALLENGE]"
RESOLUTION: "[WHAT THEY DID]"
LESSON: "The takeaway is [PRINCIPLE]"
```

## Формат ТОП-5 — Универсальная структура

**Самая мощная техника** — работает для:
- Шортсов, рилсов, длинных видео
- Подкастов, блогов, разборов
- Говорящих голов

### Трансформация тем в ТОП-5
```
"Почему нельзя покупать квартиру на 1 этаже"
→ "5 причин НЕ покупать квартиру на этом этаже"

"Как вести себя на допросе"
→ "5 правил поведения на допросе"

"Как выучить китайский"
→ "Эти 5 слов = 90% китайского языка"
```

### Полная структура ТОП-5

```
1. ОПЕНИНГ (история/статистика/вопрос)
   ↓
2. КТО Я + легкий CTA (лайк)
   ↓
3. ПОСТАНОВКА ПРОБЛЕМЫ (сторителлинг!)
   ↓
4. CTA НА ДОСМОТР ("смотрите до конца")
   ↓
5. ГЛАВА 1 [ЖИРНАЯ] ⭐
   ↓
6. CTA НА ЛАЙК
   ↓
7. ГЛАВА 2 [ЖИРНАЯ] ⭐
   ↓
8. CTA НА ПОДПИСКУ
   ↓
9. ГЛАВА 3
   ↓
10. CTA НА ПРОДУКТ (с прогревом!)
    ↓
11. ГЛАВА 4
    ↓
12. ГЛАВА 5 [В 3 РАЗА ЖИРНЕЕ!] ⭐⭐⭐
```

### Распределение ценности
```
Главы 1-2: 100% качества (жирные!)
Главы 3-4: 100% качества
Глава 5: 300% качества (в 3 раза жирнее первой!)
```

**Почему:** После первой главы — максимальный спад удержания

### Структура каждой главы

```
ШАГ 1: Название пункта
"Первое правило покупки квартиры — правильно оценить силы"

ШАГ 2: Усугубление проблемы (ДО решения!)
→ 2-3 вопроса из базовых триггеров
→ НЕ переходить сразу к решению!

ШАГ 3: Решение (20-30 секунд достаточно)

ШАГ 4: Мультиплицировать для всех глав
```

### 5 базовых триггеров для усугубления
```
1. Деньги (дёшево/дорого)
2. Время (быстрее/медленнее)
3. Обман (обманут ли?)
4. Ошибки (какие можно сделать?)
5. Своими руками (можно ли самому?)
```

### 3 типа опенинга
```
1. История: "Как-то раз всю ночь писал сценарий..."
2. Статистика: "По статистике нашей студии..."
3. Вопрос: "Как легко написать сценарий за 5 минут?"
```

### Принцип Marvel
```
Сразу после титров — экшен!
Не затягивать с представлением
```

## Квадрат Декарта (для усугублений)

| | Произойдёт | Не произойдёт |
|---|------------|---------------|
| **Будет** | Что будет? | Чего НЕ будет? |
| **Не будет** | Что будет, если НЕ? | Чего НЕ будет, если НЕ? |

Используй разные квадраты для разнообразия.

## Logic Flow Principles

### Sentence → Sentence
```
Каждая строка ведёт к следующей.
Если можно удалить без потери смысла → удали.
```

### "So What?" Chain
```
Claim: "[STATEMENT]"
So what? → "[WHY IT MATTERS]"
So what? → "[WHAT TO DO]"
So what? → "[WHAT HAPPENS IF DO/DON'T]"
```

### Setup → Payoff Balance
```
Short setup → Quick payoff (obvious points)
Long setup → Big payoff (complex reveals)

Rule: Чем больше buildup, тем больше payoff
```

---

## Buildup правило

Payoffs должны нарастать по важности:

```
Payoff 1 (хороший)
    ↓
Payoff 2 (лучше)
    ↓
Payoff 3 (ещё лучше)
    ↓
MEGA PAYOFF (самый мощный)
```

---

## Chekhov's Gun

**Правило:** Если что-то setup — должно быть payoff.

Не вводи элементы, которые не используешь. Каждый setup должен закрыться.

## Методология George Blackman

### NIPR-тест для каждой строки

> Каждая строка сценария должна содержать минимум 1 из 4 элементов. Если строка не содержит ни одного — удали её.

| Элемент | Определение | Пример |
|---------|-------------|--------|
| **N — New Information** | Даёт зрителю новую информацию | "Исследование показало 73% улучшение..." |
| **I — Story Progression** | Продвигает историю вперёд | "Когда нас забирали на следующую локацию..." |
| **P — Goal Regression** | Отдаление от цели, создаёт напряжение | "Я опаздывал. За 6 часов получил только 6 навыков..." |
| **R — Emotion** | Вызывает эмоциональную реакцию | "К этому моменту я был мёртв. Полностью истощён..." |

**Применение:**
```
Прочитай каждую строку → Проверь по NIPR → Нет ни одного? → Удали
```

---

### Transition Points — критические моменты

> Transition points — моменты перехода между сегментами. **Именно здесь зрители чаще всего уходят.**

**Правило немедленного вовлечения:**
```
Завершил сегмент → НЕ давай паузу →
Немедленно incongruous element ИЛИ следующий вопрос →
Зритель не успевает подумать об уходе
```

**Техника Incongruous Transition:**
После завершения хука — немедленно ввести что-то неожиданное:
- Неожиданная метафора или аналогия
- Визуальное изменение (смена ракурса, B-roll)
- Провокационный вопрос, не связанный напрямую

```
После хука о монетизации YouTube:
"Чтобы ответить на это, я хочу спросить вас —
вы когда-нибудь переживаете, что недостаточно видите своих друзей?"
```

---

### Payoff Placement — правило отложенного ответа

> **Проблема:** Большинство создателей дают payoff слишком рано в сегменте.

**Ошибка раннего payoff:**
```
❌ "Пункт номер один — ждать правильного сезона.
    Мы все знаем, что пастернак нуждается в идеальной погоде..."
    → Зритель уже понял суть и может пропустить
```

**Правильно — отложенный payoff:**
```
✅ "Есть кое-что, чего я хотел на YouTube больше всего.
    Но когда я это получил — моя жизнь стала более стрессовой..."
    → Постепенно раскрывайте, о чём речь
    → Зритель сам догадается и почувствует себя умнее
```

**Для образовательного контента:**
```
❌ "Номер 3: Бета-блокаторы вызывают эректильную дисфункцию"

✅ 1. Опишите симптомы, которые испытывает пациент
   2. Объясните механизм (почему это происходит)
   3. Дайте статистику (масштаб проблемы)
   4. Только потом — название препарата и решение
```

---

### Принцип повреждённого доверия

> Если payoff одного сегмента был слабым — это повреждает доверие зрителя ко ВСЕМУ видео.

```
Слабый payoff сегмента 2 →
Зритель думает: "Последние 5 минут потрачены впустую" →
Даже если сегмент 3 отличный, зритель уже настроен скептически →
Отвал на любом transition point
```

**Следствие:** Первые payoffs должны быть ЖИРНЫМИ, чтобы заработать доверие.

---

### Эмоциональные качели

> Чередование эмоциональной полярности каждые 40-60 секунд удерживает внимание.

**Паттерн качелей:**
```
ВНИЗ: Проблема, усугубление, риск
      ↓
РАЗВОРОТ: "Но...", "Однако...", "При этом..."
      ↓
ВВЕРХ: Решение, надежда, выгода
      ↓
СНОВА ВНИЗ: Нюанс, ограничение, "но есть проблема..."
      ↓
ФИНАЛЬНЫЙ ПОДЪЁМ: Итоговое решение
```

**Пример:**
```
[ВНИЗ] "90% людей совершают эту ошибку при покупке квартиры..."
[РАЗВОРОТ] "Но есть хорошая новость..."
[ВВЕРХ] "Достаточно проверить эти 3 пункта..."
[ВНИЗ] "Однако большинство риелторов вам этого не скажут..."
[ВВЕРХ] "Поэтому я подготовил чек-лист..."
```

---

### Анализ графиков удержания (Retention)

**Ключевой инсайт:** Контекст важнее изоляции

```
Неправильно: Смотреть только на отдельные провалы
Правильно: Анализировать, что зрители видели РАНЬШЕ,
           что повлияло на их поведение позже
```

**Алгоритм анализа:**
1. Смотри всё видео целиком ПЕРЕД анализом отдельных провалов
2. Ищи паттерны: провал после слабого payoff = проблема в ПРЕДЫДУЩЕМ сегменте
3. Фокусируйся на первой половине — там теряется большинство
4. **Плоские участки важнее провалов** — изучай, почему зрители оставались
5. Transition points = главные точки риска

---

### Ясность структуры для аудитории

> Не просто знай структуру сам — **делай её кристально ясной для зрителя.**

**Почему критично:**
- Зритель должен знать, где он находится в видео
- Ясная структура снижает когнитивную нагрузку
- Предсказуемость формата увеличивает комфорт просмотра

**Техники визуализации:**
```
В начале: "Сегодня разберём 5 пунктов..." + визуальные маркеры
На каждом transition: Чёткое обозначение перехода
Прогресс-бары или счётчики (особенно для listicles)
```

## Anti-patterns переходов

**НЕ ДЕЛАТЬ:** Дублирование вывода и начала следующей секции

```
❌ ПЛОХО:
Section 2: "...The truth is more nuanced."
Section 3: "Here is something fascinating. The cancer protection..."

✅ ХОРОШО:
Section 2: "...The truth is more nuanced."
Section 3: "The cancer protection from coffee is probably not from caffeine."
```

**Правило:** Если секция заканчивается выводом/zinger, следующая секция НЕ начинается с filler-фразы. Сразу в суть следующего тезиса.

**Типичные filler-фразы (избегать):**
- "Here is something fascinating"
- "Let me tell you something interesting"
- "Now here is the thing"
- "This is where it gets interesting"

## Чек-лист structure

- [ ] Логлайн написан до сценария
- [ ] Каждый payoff 300-500 слов
- [ ] Logic flow от предложения к предложению
- [ ] "So what?" отвечен для каждого claim
- [ ] Specific примеры и цифры
- [ ] Payoffs нарастают к MEGA PAYOFF
- [ ] Все setups имеют payoffs
- [ ] **Нет дублирования переходов между секциями**
