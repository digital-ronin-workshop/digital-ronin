# Методы извлечения смыслов

> Смысл — это не то, что ты говоришь о продукте. Это то, что клиент понимает о своей жизни.

---

## Value Proposition Canvas

Инструмент Остервальдера для создания ценностного предложения.

### Профиль клиента (Customer Profile)

| Компонент | Вопрос | Пример (курс копирайтинга) |
|-----------|--------|----------------------------|
| **Jobs** | Что клиент пытается сделать? | Зарабатывать на текстах, найти удалённую работу |
| **Pains** | Что мешает, раздражает, пугает? | Низкие цены, много конкурентов, нет клиентов |
| **Gains** | Что хочет получить, о чём мечтает? | Стабильный доход 150К+, свобода, признание |

### Карта ценности (Value Map)

| Компонент | Вопрос | Пример |
|-----------|--------|--------|
| **Products & Services** | Что предлагаем? | Курс + менторство + сообщество |
| **Pain Relievers** | Как снимаем боли? | Готовые скрипты, портфолио, связи с заказчиками |
| **Gain Creators** | Как создаём выгоды? | Методика быстрого роста цен, личный бренд |

### Шаблон заполнения

```markdown
## Value Proposition: [Продукт]

### Профиль клиента
**Jobs (что делает):**
1. [основная задача]
2. [дополнительная задача]
3. [эмоциональная задача]

**Pains (что мешает):**
1. [функциональная боль]
2. [эмоциональная боль]
3. [социальная боль]

**Gains (чего хочет):**
1. [обязательный результат]
2. [ожидаемый результат]
3. [неожиданный бонус]

### Карта ценности
**Продукт/услуга:**
- [компонент 1]
- [компонент 2]

**Как снимаем боли:**
- [боль 1] → [как снимаем]
- [боль 2] → [как снимаем]

**Как создаём выгоды:**
- [выгода 1] → [как создаём]
- [выгода 2] → [как создаём]

### ИТОГОВЫЙ СМЫСЛ
> [1 предложение, объединяющее главную боль и главную выгоду]
```

## Jobs-to-be-Done (JTBD)

> Клиенты не покупают продукты — они нанимают их для выполнения работы.

### Три слоя работы

| Слой | Вопрос | Пример (фитнес-курс) |
|------|--------|----------------------|
| **Функциональная** | Что нужно сделать? | Похудеть на 10 кг |
| **Эмоциональная** | Как хочу себя чувствовать? | Уверенно, энергично |
| **Социальная** | Как хочу выглядеть в глазах других? | Привлекательно, здорово |

### Job Story формат

```
Когда [ситуация/контекст],
я хочу [мотивация/действие],
чтобы [ожидаемый результат].
```

**Примеры:**

**Онлайн-курс:**
```
Когда я готовлюсь сменить профессию,
я хочу получить востребованный навык за 3 месяца,
чтобы не тратить годы на обучение и начать зарабатывать быстрее.
```

**B2B-сервис:**
```
Когда моя команда тонет в рутинных задачах,
я хочу автоматизировать повторяющиеся процессы,
чтобы сотрудники занимались тем, что приносит деньги.
```

**Консалтинг:**
```
Когда я застрял на одном уровне дохода,
я хочу получить взгляд со стороны от того, кто уже прошёл этот путь,
чтобы не тратить год на ошибки, которые можно избежать за месяц.
```

## Pain-Gain Mapping

### Процесс извлечения смысла

```
БОЛЬ (что не работает)
    │
    ├──→ Последствия (что теряет)
    │
    ├──→ Эмоция (что чувствует)
    │
    └──→ ЖЕЛАНИЕ (чего хочет вместо этого)
              │
              ├──→ Результат (что получит)
              │
              ├──→ Механизм (как это работает)
              │
              └──→ СМЫСЛ (1 предложение)
```

### Примеры маппинга

**Пример 1: Курс по продажам**
```
БОЛЬ: Клиенты уходят "подумать" и не возвращаются
  ├─ Последствия: Потеря 70% потенциальных сделок
  ├─ Эмоция: Разочарование, неуверенность
  └─ ЖЕЛАНИЕ: Закрывать сделки на первой встрече
        ├─ Результат: Конверсия 50%+ вместо 10%
        ├─ Механизм: Техника "3 вопроса перед возражением"
        └─ СМЫСЛ: "Клиенты перестанут уходить думать —
                   они будут покупать на месте"
```

**Пример 2: SaaS для бизнеса**
```
БОЛЬ: Менеджеры тратят 5 часов в день на рутину
  ├─ Последствия: -100К в месяц на каждого сотрудника
  ├─ Эмоция: Выгорание, текучка кадров
  └─ ЖЕЛАНИЕ: Автоматизировать рутину
        ├─ Результат: Освободить 4 часа в день
        ├─ Механизм: ИИ-ассистент для типовых задач
        └─ СМЫСЛ: "Ваши менеджеры займутся продажами,
                   а не заполнением таблиц"
```

**Пример 3: Личный бренд**
```
БОЛЬ: Много работы, мало признания и денег
  ├─ Последствия: Застревание на одном уровне годами
  ├─ Эмоция: Обесценивание, усталость
  └─ ЖЕЛАНИЕ: Чтобы клиенты приходили сами
        ├─ Результат: Очередь из клиентов, цены x3
        ├─ Механизм: Экспертный контент + позиционирование
        └─ СМЫСЛ: "Станьте тем, к кому обращаются
                   без сравнения с конкурентами"
```

## Вопросы для извлечения смыслов

### Про проблему
1. Что не получается у клиента прямо сейчас?
2. Что он уже пробовал и почему не сработало?
3. Сколько это стоит ему в деньгах/времени/нервах?
4. Как долго он живёт с этой проблемой?
5. Что будет, если ничего не менять?

### Про желание
1. Как выглядит идеальный результат?
2. Что изменится в жизни, когда проблема решена?
3. Как он поймёт, что цель достигнута?
4. О чём он мечтает, но боится признаться?
5. Кем он хочет стать?

### Про решение
1. Почему ваш метод работает, а другие — нет?
2. Что уникального в вашем подходе?
3. Какой минимальный результат гарантируете?
4. Как быстро клиент увидит первые изменения?
5. Почему именно сейчас лучшее время начать?

## Формула извлечения

```
СМЫСЛ = [Боль] + [Механизм] + [Результат]
```

**Шаблон:**
```
"[Целевая аудитория] перестанут [боль],
потому что [механизм],
и получат [конкретный результат]"
```

**Примеры:**
- "Эксперты перестанут продавать за копейки, потому что освоят технику ценностного позиционирования, и поднимут цены в 3 раза без потери клиентов"
- "Предприниматели перестанут тонуть в операционке, потому что внедрят систему делегирования, и освободят 20 часов в неделю для стратегии"

## Чек-лист качества смысла

- [ ] Боль узнаваема (клиент скажет "это про меня")
- [ ] Последствия конкретны (цифры, сроки)
- [ ] Желание достижимо (не фантастика)
- [ ] Механизм понятен (не магия)
- [ ] Результат измерим (как проверить?)
- [ ] Смысл умещается в 1 предложение
