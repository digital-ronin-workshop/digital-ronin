# Tripwire (Продукт-пробник)

> Источник: Методология Сердитова

## Что это

**Tripwire** — недорогой продукт (2-5к), который:
1. Устанавливает первый платёжный контакт
2. Даёт мощный вау-эффект
3. Прогревает к покупке основного продукта

---

## Зачем нужен

```
Холодный трафик → Tripwire → Основной продукт

НЕ:
Холодный трафик → Основной продукт за 100к (не работает)
```

**Логика клиента:**
```
"Если мне за 3к дали ТАКОЕ,
то что будет за 100к?!"
```

---

## Чего НЕ должен делать tripwire

❌ Полностью решать проблему клиента
❌ Быть самостоятельным продуктом
❌ Конкурировать с основным продуктом

**Ошибка Сердитова:**
Продавал tripwire за 3к как самостоятельный продукт.
Заработал 13 млн. Но если бы вёл на основной — было бы под 100 млн.

---

## Что ДОЛЖЕН делать tripwire

✅ Ввести в курс дела
✅ Объяснить КАК и ЗА СЧЁТ ЧЕГО решается проблема
✅ Показать, что основной продукт — это решение
✅ Создать вау-эффект
✅ Прогреть к покупке основного

---

## Структура tripwire

### Формат
- 4-5 уроков
- Короткие (не часовые вебинары)
- Каждый урок = одна ценная идея

### Как создать темы уроков

**Шаг 1:** Взять обещания из распаковки (самые ключевые)

**Шаг 2:** Сократить до названия урока

**Шаг 3:** Название должно звучать как отдельный продукт

### Пример трансформации

**Обещание (длинное):**
```
Ты внедришь дополнительную услугу микроворонка 
и минимум в 3 раза увеличишь средний чек...
```

**Название урока:**
```
"Что такое микроворонка и как она увеличит 
твой средний чек минимум в 3 раза?"
```

---

## Примеры названий уроков

### Ниша: Таргет

1. "Что такое микроворонка и как она увеличит твой чек минимум в 3 раза?"
2. "Как приносить клиентам результат в 90% случаев, тратя в 3 раза меньше времени?"
3. "Лёгкие способы находить клиентов из самых прибыльных ниш с чеком от 30к"
4. "Как перестать быть фрилансером и стать партнёром за процент от продаж"
5. "Как создать упакованный блог, который отрабатывает возражения на входе"

### Правило названий
Каждое название должно звучать так, 
как будто продаёшь не урок, а отдельный продукт.

---

## Буллеты

**Что это:** Дополнительные тезисы, которые усиливают описание.

**Откуда брать:** Из содержания уроков — выделить отдельные инсайты.

### Пример

**Урок:** "Что такое микроворонка..."

**Буллет:**
```
"Я расскажу секрет тех самых таргетологов, 
которые получают от 200к за один проект, 
и покажу как это делать тебе"
```

(Этот инсайт прозвучит в уроке, но можно выделить отдельно)

---

## Оффер tripwire

### Формула 1: Интрига
```
Узнай как [оффер основного продукта]
```

**Пример:**
```
Основной: "Зарабатывать 150к в месяц..."
Tripwire: "Узнай как зарабатывать от 150к в месяц..."
```

### Формула 2: Элементы
```
5 шагов/элементов, которые позволят тебе [результат]
```

**Пример:**
```
"5 простых элементов, которые позволят тебе 
зарабатывать от 150к ежемесячно, 
освоив технологию привлечения для блогеров и инфобиза"
```

## Воронка с tripwire

```
Трафик (реклама/контент)
         ↓
Статья (прогрев + оффер tripwire)
         ↓
Tripwire (3к) — вау-эффект
         ↓
Дожим на основной продукт (внутри tripwire)
         ↓
Основной продукт (100к)
```

## Чек-лист tripwire

- [ ] Цена 2-5к (легко расстаться)
- [ ] 4-5 уроков максимум
- [ ] Каждый урок = продающее название
- [ ] Даёт вау-эффект
- [ ] НЕ решает проблему полностью
- [ ] Показывает, что решение = основной продукт
- [ ] Есть буллеты для усиления
- [ ] Есть оффер (интрига или элементы)

## Отбор обещаний для уроков

*Источник: Сердитов, курс БЗЧ — "Продукт пробник"*

### [TECHNIQUE] Фильтрация обещаний по ядру желания

Не все обещания из распаковки годятся для tripwire. Критерий отбора:

**Берём** — обещания, направленные на ЯДРО желания (деньги, результат, трансформация).

**Отсеиваем:**
- Опции продукта ("получишь доступ к базе кейсов") — это фича, не желание
- Второстепенные навыки ("не будешь сталкиваться с блокировками") — это не то, ради чего покупают

```
ФИЛЬТР: Спроси себя — человек ПРИШЁЛ ради этого?
         Или это просто приятный бонус?

Пришёл ради → берём в урок
Приятный бонус → отсеиваем
```

### [EXAMPLE] Отбор для ниши "Таргетолог"

| Обещание | Решение | Причина |
|----------|---------|---------|
| "Внедришь микроворонку, увеличишь чек в 3 раза" | БЕРЁМ | Направлено на деньги |
| "В 90% случаев будешь выполнять обязательства" | БЕРЁМ | Направлено на результат |
| "Находить клиентов из прибыльных ниш с чеком от 30к" | БЕРЁМ | Направлено на деньги |
| "Получишь доступ к котлу с кейсами" | ОТСЕИВАЕМ | Это опция, не желание |
| "Не будешь сталкиваться с блокировками" | ОТСЕИВАЕМ | Второстепенный навык |

## Названия уроков: продающая формулировка

*Источник: Сердитов, курс БЗЧ — "Продукт пробник"*

### [TECHNIQUE] Критериально-измеримый результат в названии

Каждое название урока ОБЯЗАНО содержать конкретную цифру или измеримый результат. Без этого название "мёртвое".

```
ФОРМУЛА НАЗВАНИЯ:
[Как/Что такое] + [суть] + [измеримый результат с цифрой]
```

### [ANTI-PATTERN] Генерик-названия уроков

```
ПЛОХО:
- "Урок 1: Микроворонка"
- "Урок 2: Поиск клиентов"
- "Урок 3: Упаковка аккаунта"

Реакция: "Ну, поиск клиентов. И что с этим делать?"

ХОРОШО:
- "Что такое микроворонка и как она увеличит твой чек минимум в 3 раза?"
- "Лёгкие способы находить клиентов из самых прибыльных ниш с чеком от 30к"
- "Как создать упакованный блог, который на входе отрабатывает все возражения"

Реакция: "О, хочу открыть!"
```

### [PATTERN] Тест на название урока

Название должно проходить проверку:

> "Если бы это был не урок, а ОТДЕЛЬНЫЙ платный продукт — купили бы по этому названию?"

Да → название годится.
Нет → переписать.

## Два формата структуры tripwire

*Источник: Сердитов, курс БЗЧ — "Продукт пробник"*

### [PATTERN] Элементы системы vs Пошаговая схема

Tripwire строится в одном из двух форматов:

**Формат 1: Элементы системы**
```
"5 элементов, которые позволят тебе [результат]"
Каждый урок = один элемент системы
```

**Формат 2: Пошаговая схема**
```
"Пошаговая схема, как [результат]"
Каждый урок = один шаг
```

Выбор формата зависит от продукта:
- Если результат достигается КОМБИНАЦИЕЙ компонентов → элементы системы
- Если результат достигается ПОСЛЕДОВАТЕЛЬНОСТЬЮ действий → пошаговая схема

## Техника формирования буллетов

*Источник: Сердитов, курс БЗЧ — "Продукт пробник"*

### [TECHNIQUE] Извлечение буллетов из содержания уроков

Буллеты — это НЕ названия уроков. Это побочные тезисы, которые звучат в уроках, но вынесены отдельно для усиления.

**Метод извлечения:**

```
Шаг 1: Записать урок (или составить содержание)
Шаг 2: Переслушать/перечитать
Шаг 3: Найти тезисы, которые НЕ вынесены в название урока
Шаг 4: Сформулировать как отдельный буллет
```

### [EXAMPLE] Буллет из урока про микроворонку

```
Урок: "Что такое микроворонка и как она увеличит чек в 3 раза?"

Внутри урока звучит: "Микроворонку можно продавать
в связке с трафиком за 200 тысяч рублей за проект"

Извлечённый буллет:
"Я расскажу секрет тех самых таргетологов,
которые получают от 200к за один проект,
и покажу как это делать тебе"
```

### [EXAMPLE] Буллет из урока про результаты

```
Урок: "Как приносить клиентам результат в 90% случаев..."

Внутри урока звучит: "Результат достигается за счёт
готовых связок, которые настраиваются за 20 минут"

Извлечённый буллет:
"Ты узнаешь, что такое готовые связки, которые позволяют
настраивать рекламу для любого клиента за 20 минут"
```

### [PATTERN] Размещение буллетов в продающей статье

```
БЛОК ОПИСАНИЯ ПРОДУКТА В СТАТЬЕ:

1. Перечисление уроков (с продающими названиями)
2. Строка-переход: "А ещё:"
3. Буллеты (дополнительные тезисы)
   - "Ты узнаешь..."
   - "Я покажу..."
   - "Расскажу секрет..."
```

## Оффер пробника: техника интриги

*Источник: Сердитов, курс БЗЧ — "Продукт пробник"*

### [TECHNIQUE] Замена результативного глагола на интригу

Берём оффер основного продукта и меняем только глагол:

```
ОСНОВНОЙ: "Ты НАУЧИШЬСЯ зарабатывать 150к в месяц..."
TRIPWIRE: "УЗНАЙ КАК зарабатывать 150к в месяц..."

Результативный глагол → Глагол интриги
"научишься" → "узнай как"
"внедришь"  → "узнай как"
"создашь"   → "узнай как"
"будешь"    → "узнай как"
```

### [ANTI-PATTERN] Копирование оффера основного продукта без изменения

```
ОШИБКА:
Основной: "Ты научишься зарабатывать 150к в месяц"
Tripwire: "Ты научишься зарабатывать 150к в месяц"

ПОЧЕМУ ПЛОХО: В коротком дешёвом продукте невозможно
выполнить это обещание. Подорвёт доверие.

ПРАВИЛЬНО:
Tripwire: "Узнай КАК зарабатывать от 150к в месяц"
(интрига вместо гарантии результата)
```

## Бонусный банк

*Источник: Сердитов, курс БЗЧ — "Продукт пробник"*

### [TECHNIQUE] Единый банк бонусов

Все придуманные бонусы хранятся в ОДНОМ месте, а не распределяются сразу по продуктам.

```
Шаг 1: Сгенерировать ВСЕ бонусы, которые можно придумать
Шаг 2: Хранить в едином банке
Шаг 3: Распределять между продуктами позже —
        когда пишете продающую статью

ПОЧЕМУ: На этапе создания tripwire вы ещё не знаете,
какой бонус лучше отдать tripwire, а какой — основному.
Решение принимается при сборке продающей статьи.
```

## Расширенный чек-лист tripwire

*Источник: Сердитов, курс БЗЧ — "Продукт пробник"*

### [CHECKLIST] Полная проверка tripwire

**Структура:**
- [ ] Выбран формат: элементы системы ИЛИ пошаговая схема
- [ ] 4-5 уроков (не больше)
- [ ] Обещания отфильтрованы по ядру желания (не опции, не второстепенное)
- [ ] Каждое название урока содержит измеримый результат с цифрой
- [ ] Каждое название звучит как отдельный продукт (тест: "купили бы?")
- [ ] Нет генерик-названий ("Поиск клиентов", "Микроворонка")

**Буллеты:**
- [ ] Извлечены из содержания уроков (побочные тезисы)
- [ ] Формат: "Ты узнаешь..." / "Я покажу..." / "Расскажу секрет..."
- [ ] Размещение в статье: после уроков, через "А ещё:"

**Оффер:**
- [ ] Результативный глагол заменён на интригу ("узнай как")
- [ ] НЕ копирует оффер основного продукта дословно
- [ ] Один из двух форматов: "Узнай как [результат]" или "N элементов для [результат]"

**Бонусы:**
- [ ] Все бонусы в едином банке
- [ ] Распределение между продуктами — при написании статьи, не раньше
