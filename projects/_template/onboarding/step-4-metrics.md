# Шаг 4: Сбор метрик и обучение

> Публикация — это не конец. Это начало обратной связи.

---

## Learning Loop

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  PUBLISH     │ ──→ │  48h WAIT    │ ──→ │  COLLECT     │
│  (вручную)   │     │  (метрики)   │     │  (analysis)  │
└──────────────┘     └──────────────┘     └──────┬───────┘
                                                  │
                                                  ↓
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  IMPROVE     │ ←── │  ANALYZE     │ ←── │  RECORD      │
│  (playbook)  │     │  (паттерны)  │     │  (метрики)   │
└──────────────┘     └──────────────┘     └──────────────┘
```

---

## Какие метрики собирать

| Метрика | Где смотреть | Важность |
|---------|--------------|----------|
| Views | Threads app | High |
| Likes | Threads app | High |
| Comments | Threads app | Medium |
| Saves | Threads app | **High** (сигнал намерения) |
| Переходы в TG | Telegram stats | **High** (воронка) |
| ER% | (likes+comments+saves)/views | High |

---

## Когда собирать

**Через 48 часов после публикации.**

Раньше — данные неполные.
Позже — можно забыть.

---

## Команда сбора

```
metrics [дата]
```

Создаёт шаблон для записи метрик по дате.

---

## Формат записи

**Файл:** `analysis/YYYY-MM-DD-metrics.md`

```markdown
# Метрики — [дата]

> **Записано:** [дата + 48ч]

## Сводка

| # | Тема | Формат | Views | Likes | Comments | Saves | ER% |
|---|------|--------|-------|-------|----------|-------|-----|
| 1 | [тема] | [формат] | X | X | X | X | X% |

## Детальный разбор

### Post 1: [тема]

**Pipeline Score:** X%
**Actual ER:** X%

**Почему сработало / не сработало:**
- [анализ]

**Паттерн для повторения:**
- [если успех]
```

---

## Что анализировать

### Top 3 (лучшие по ER%)
- Какая тема?
- Какой формат?
- Какой hook?
- Что общего?

### Bottom 3 (худшие по ER%)
- Почему не сработало?
- Что исправить?
- Добавить в антипаттерны?

---

## Еженедельный review

**Когда:** Каждое воскресенье

**Процесс:**
1. Собрать все метрики недели
2. Определить Top 3 и Bottom 3 по ER%
3. Выявить паттерны успеха/неудачи
4. Обновить `threads-playbook.md`
5. Скорректировать format_mix для следующей недели

## Корреляция Score ↔ ER

Отслеживай со временем:

| Pipeline Score | Фактический ER |
|----------------|----------------|
| 90%+ | → обычно 3%+ |
| 70-90% | → обычно 1.5-3% |
| <70% | → обычно <1.5% |

Если корреляция слабая — нужно улучшить аватары.
