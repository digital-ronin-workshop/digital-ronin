# AI-автоматизация исследований (связка нейросетей)

> Как сократить время работы с CustDev и дашбордом, используя AI Touch + MeetMyAI.

---

## Общая связка

### [FORMULA] AI-конвейер для исследования

```
1. Выявление сегмента   →  AI Touch (Jobs To Be Done анализ)
2. Приглашение на CustDev →  Контент по сегментам из AI
3. Созвон + запись       →  Диктофон / Telegram
4. Транскрибация         →  MeetMyAI (авто-транскрипт + AI-отчёт)
5. Анализ транскрипта    →  AI Touch (извлечение болей, желаний, возражений)
6. Заполнение дашборда   →  Копи-паст из AI-ответов
```

---

## Шаг 1: Выявление сегмента через AI Touch

### [TECHNIQUE] Jobs To Be Done анализ через AI

**Платформа:** AI Touch (GPT-4 Box)

**Промпт (часть 1):**
```
Проведи Jobs To Be Done анализ для проекта [описание проекта].
[Описание: какой результат хочет получить клиент]

Проведи сегментацию потенциальных клиентов,
пропиши их боли, потребности и предложи релевантное решение.

В виде подробной таблицы опиши все сегменты и их JTBD:
- Какую работу для них нужно выполнить
- Какой конечный результат можно дать
- Боли
- Потребности
- Решение
- Темы для контента (для привлечения в соцсетях)
```

**Результат:** Таблица с 4-6 сегментами, их JTBD, болями, контент-темами.

**Промпт (часть 2):**
```
Добавь несколько неочевидных сегментов аудитории,
которым могут быть полезны мои услуги.
```

→ Расширяет список на 3-5 неочевидных сегментов.

**Промпт (часть 3-4, факультативно):**
```
Объедини все столбцы в одну таблицу
и на основе этого создай подробный гайд по позиционированию.
```

### [PATTERN] Как использовать результат
- Сегменты → Шаг 1 дашборда (широкий + узкий сегмент)
- JTBD → Шаг 2 дашборда (зачем клиенту нужен результат)
- Темы для контента → Stories/Reels для привлечения конкретного сегмента на CustDev

## Шаг 2: Транскрибация созвона через MeetMyAI

### [TECHNIQUE] Авто-транскрибация CustDev

**Платформа:** MeetMyAI

**Процесс:**
1. Созвон с респондентом (Telegram/Zoom)
2. Параллельно — запись на диктофон (iPhone рядом)
3. Загрузить аудио в MeetMyAI → "Добавить встречу" → "Загрузить файл"
4. 5-10 минут обработки
5. Получаешь: полный транскрипт + AI-отчёт (краткое содержание) + задачи

**Формат скачивания:** .docs (лучше для AI Touch)

## Шаг 3: Извлечение данных из транскрипта через AI Touch

### [TECHNIQUE] Автоизвлечение болей и желаний

**Загрузить .docs файл в AI Touch и скармливать промпты по шагам дашборда:**

**Промпт для болей (Шаг 1 дашборда):**
```
Действуй как маркетолог.
Я высылаю файл транскрипции созвона с респондентом (юзер-ресерч).

Первым шагом вынь из диалога:
что в данный момент у него не так, что не устраивает,
от чего плохо в жизни/работе — касательно темы [ваша тема].

Формулируй в формате:
"У меня нет..."
"Мне плохо от..."
"Мне некомфортно..."
"Мне неудобно..."
```

**Промпт для желаний:**
```
Теперь найди ответы на вопрос:
"А как ты хочешь, чтобы стало?"
```

**Промпт для ожиданий от продукта (Шаг 2):**
```
Выяви ожидания клиента от продукта.
Узнай, каким он должен быть — понятным, критериально измеримым.
```

**Промпт для обещаний (Распаковка):**
```
Подумай, как удовлетворить эти желания
и превратить их в конкретные обещания.

Формула: "Ты" + глагол обещания результата +
описание того, каким образом это обещание будет реализовано.

Пример: [вставь пример из своего дашборда]
```

### [PATTERN] Результат копируется прямо в дашборд
- AI даёт дословные цитаты из созвона
- Группирует по темам (питание, мотивация, спорт и т.д.)
- Каждый следующий промпт — следующий шаг дашборда

---

## Ключевые инсайты

### [PATTERN] AI не заменяет CustDev, а ускоряет обработку
- Всё равно нужно проводить реальные созвоны
- AI экономит часы на пересмотре записей и заполнении таблиц
- Связка позволяет обрабатывать 5-10 CustDev за день вместо одного

### [MEANING] Шаги 5 и 6 дашборда = деньги
> "Только шаги 5 и 6 позволили мне сделать 2 продажи на 1.5 млн рублей"
Не пропускайте последние шаги — именно там конверсия в продажу.
