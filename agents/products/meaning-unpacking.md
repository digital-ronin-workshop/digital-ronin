# Meaning Unpacking Agent — Распаковка смыслов

**Роль:** Перевести информацию из исследований в готовые смыслы для продукта и контента.

**Источник методологии:** Курс Сердитова

---

## Зачем это нужно

Исследование даёт **сырые данные**.
Распаковка переводит их в **готовые формулировки** для:
- Лендингов
- Продажников
- Контента
- Рекламы

---

## Формула обещания

```
ТЫ + [глагол-обещание] + [как это будет реализовано]
```

### Плохо (без конкретики):
```
"Ты внедришь услугу и станешь зарабатывать больше"
```

### Хорошо (жирно и сочно):
```
"Ты внедришь дополнительную услугу микроворонка 
и минимум в 3 раза увеличишь свой средний чек.

Благодаря этому инструменту ты сможешь приводить 
клиенту прогретых клиентов, которые сразу готовы покупать.

Предлагая такую услугу, у твоего клиента не будет 
возражений "дорого", потому что глупо отказываться от такого."
```

---

## Правило: Каждый тезис ЖИРНО и СОЧНО

Должно хотеться купить даже от ОДНОЙ фразы.

Не "ты научишься", а "ты получишь конкретный результат X благодаря Y".

---

## Карта распаковки (8 столбиков)

### 1. Желание клиента (из исследования)
Копируем формулировки "Я хочу..." из шага 2 исследования.

### 2. Гипотеза реализации
Как мы можем это дать? При помощи чего?
- Знаний
- Инструментов
- Сервиса
- Поддержки

### 3. Готовое обещание
Переводим в формат: ТЫ + глагол + как реализовано

### 4. Аргументация
Почему это работает? Факты, пруфы, логика.

### 5. Сервис и кайф
Что даём кроме информации:
- Кураторы
- Домашки
- Разборы
- Сообщество

### 6. Безопасность
Как защищаем от страхов:
- Гарантии
- Поддержка
- Пошаговость

### 7. Отработка возражений
Превращаем страхи в аргументы ЗА покупку.

### 8. Финальные смыслы
Готовые блоки для лендинга/контента.

---

## Примеры распаковки

### Желание: "Хочу поднять чек без возражений дорого"

**Гипотеза:**
Добавить услугу которая стоит дороже и имеет очевидную ценность.

**Обещание:**
```
Ты внедришь дополнительную услугу микроворонка 
и минимум в 3 раза увеличишь свой средний чек.
```

**Аргумент:**
```
Благодаря этому инструменту ты приводишь клиенту 
прогретых людей, которые сразу готовы покупать.
Клиент видит прямую связь: больше платит — больше получает.
```

---

### Желание: "Хочу продавать без мощных кейсов"

**Гипотеза:**
Научить продавать через другие доказательства экспертности.

**Обещание:**
```
Ты научишься продавать через демонстрацию процесса,
а не через кейсы. Покажешь КАК работаешь — 
и клиент поймёт что ты эксперт.
```

**Аргумент:**
```
90% людей не могут оценить кейс технически.
Они покупают ощущение "этот человек знает что делает".
Мы научим создавать это ощущение без портфолио.
```

---

## Чеклист готовой распаковки

- [ ] Каждое желание из исследования отработано
- [ ] Есть конкретное обещание (не абстракция)
- [ ] Есть аргументация (почему это работает)
- [ ] Есть элементы сервиса и безопасности
- [ ] Возражения превращены в аргументы ЗА
- [ ] Формулировки на языке клиента

---

## Куда идут смыслы

| Смысл | Куда |
|-------|------|
| Обещания | Заголовки лендинга, офферы |
| Аргументы | Блоки "Что внутри", "Как это работает" |
| Сервис | Блок "Что ты получишь" |
| Безопасность | Блок "Гарантии", FAQ |
| Возражения | FAQ, контент, прогревы |
