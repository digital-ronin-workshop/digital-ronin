## TRIGGERS

Активируется на простые слова. После триггера уточняет детали.

| Триггер | Что делает | Уточняющие вопросы |
|---------|------------|-------------------|
| `продающие смыслы` | **Главный триггер** — полный цикл работы со смыслами | Для чего? Какой проект? |
| `смысл` | Извлечь или создать продающий смысл | Для чего? Какая ЦА? |
| `продающий` | Усилить текст/идею | Что усилить? Для кого? |
| `почему купят` | Найти главную причину покупки | Какой продукт? |
| `боль аудитории` | Выявить боли ЦА | Какой проект/продукт? |
| `усиль` | Применить триггеры к тексту | Какой текст? |

### Поведение после триггера

1. Уточнить продукт/нишу
2. Загрузить контекст проекта (если указан)
3. Применить методологию
4. Выдать результат + предложить усиление

## GENERATION MODES

### `/meanings extract [продукт/ниша]`

Извлечь ядро смысла из продукта или ниши.

**Input:** Описание продукта, ЦА, проблемы

**Process:**
1. Загрузить skill/extraction.md
2. Применить Value Proposition Canvas
3. Построить Pain-Gain маппинг
4. Сформулировать ядро смысла

**Output:**
```markdown
## Extraction: [Продукт]

### БОЛЬ
- Что не работает: [конкретная проблема]
- Последствия: [что теряет клиент]
- Эмоция: [что чувствует]

### ЖЕЛАНИЕ
- Результат: [что хочет получить]
- Трансформация: [как изменится жизнь]
- Эмоция: [что хочет чувствовать]

### МЕХАНИЗМ
- Как это работает: [почему ваше решение]
- Уникальность: [чем отличается от других]

### ЯДРО СМЫСЛА
> [1 предложение, объединяющее боль + механизм + результат]

### ТРИГГЕРЫ
| Триггер | Как применить |
|---------|---------------|
| [триггер 1] | [формулировка] |
| [триггер 2] | [формулировка] |
```

## KEY METHODOLOGIES

### Метод скользкой горки (главный принцип)

```
Скользкая горка Халилова/Шугермана:
"Задача первого предложения — заставить прочитать второе"

Каждый элемент текста передаёт "эстафету любопытства":
Заголовок → Первая фраза → Каждый абзац → CTA

Читатель не может остановиться — как санки на горке.
```

**Элементы горки:**
| Элемент | Назначение |
|---------|------------|
| Сцепление | Первая фраза цепляет и не отпускает |
| Интрига | Открытая петля, которую хочется закрыть |
| Эстафета | Каждое предложение → следующее |
| Твист | Неожиданный поворот для внимания |
| Конкретика | Цифры, факты, имена (за что цепляется мозг) |

### Процесс создания смысла

```
1. RESEARCH
   └─ Загрузить project/brand/audience.md (боли ЦА)
   └─ Понять контекст и проблему
   └─ Определить этап CJM клиента

2. EXTRACT
   └─ Применить JTBD / Value Proposition
   └─ Найти ядро: боль + желание + механизм

3. FORMULATE
   └─ Сжать до 1 предложения
   └─ Проверить: узнаваемо? конкретно? уникально?
   └─ Применить правило "можно рассказать другу за 30 сек"

4. AMPLIFY
   └─ Добавить триггер (из psychology.md)
   └─ Усилить конкретикой (цифры, факты, имена)
   └─ Встроить в скользкую горку

5. DISTRIBUTE
   └─ Адаптировать под формат
   └─ Сохранить ядро, изменить подачу
   └─ Проверить "упругость" — убрать всё лишнее
```

### Формула ядра смысла

```
СМЫСЛ = [Боль] + [Механизм] + [Результат]

"[ЦА] перестанут [боль],
потому что [механизм],
и получат [результат]"
```

## ANTI-PATTERNS (Never do)

| Плохо | Почему | Как правильно |
|-------|--------|---------------|
| Абстрактные выгоды | "Станет лучше" — пусто | Конкретный результат с цифрой |
| Много смыслов сразу | Размывает фокус | Один смысл на единицу контента |
| Смысл о продукте | Людям всё равно о продукте | Смысл о жизни клиента |
| Копирование конкурентов | Все одинаковые | Найти уникальный угол |
| Игнорирование возражений | Клиент уйдёт думать | Встроить ответ в смысл |

## QUALITY CHECKLIST

Перед выдачей смысла проверить:

### Ядро смысла
- [ ] Умещается в 1 предложение?
- [ ] Боль узнаваема (клиент скажет "это про меня")?
- [ ] Результат конкретен и измерим?
- [ ] Есть психологический триггер?
- [ ] Понятно, почему действовать сейчас?
- [ ] Отличается от конкурентов?
- [ ] Можно адаптировать под разные форматы?

### Скользкая горка (по Халилову)
- [ ] Первое предложение заставляет читать второе?
- [ ] Есть "эстафета любопытства" между частями?
- [ ] Текст упругий (нечего убрать без потери смысла)?
- [ ] Есть конкретика (цифры, факты, имена)?
- [ ] Можно пересказать другу за 30 секунд?
- [ ] Главный герой — читатель (не продукт)?
- [ ] Нет канцеляризмов и воды?
- [ ] Занята чёткая позиция (за белых или за красных)?
